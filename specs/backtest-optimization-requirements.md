## Feature Description

Optimize the crypto-futures backtesting system from multi-hour to seconds-to-minutes (TradingView-class) WITHOUT changing business logic (<1% deviation from real trading; golden-master bit-identical on the canonical 5m no-drill path). Rollout in 7 phases (P0 golden-master harness → P1 cache fix → P2 loaders/sweep → P3 SoA engine → P4 numba → P5 Parquet/DuckDB → P6 vectorized fast-path).

**Total curated requirements: 517** (must/should/could split in summary below). **Plus two externally-sourced /
backported amendment sets counted SEPARATELY from the 517 (mirrored in spec §A.1/§Y): REQ-SEC-001..007 (origin:
architecture §8.7) and REQ-PERF-046 (backported from spec §I.0a — numba/pure-Python lane split + committed
pure-Python budget).**

## Parity-Correctness [PAR] — 45 items

- **REQ-PAR-001** _(must, P0)_ — Intrabar exit precedence stays liquidation→SL→TP, pessimistic: on dual SL+TP touch SL wins, SL fires only if closer than the liq price, and the existing `>0` PnL-fabrication guards are retained.
- **REQ-PAR-002** _(must, P0)_ — Within-candle rule order is fixed funding→per-position liq/TP/SL→equity cascade→trailing→time; ALL per-position exits resolve before the basket cascade, and a position's own liq/TP/SL beats a same-bar basket flatten (records TP/SL/liquidation, not equity_*).
- **REQ-PAR-003** _(must, P0)_ — Equity cascade order EQUITY_DROP(/SMART)→close_on_profit→EQUITY_RISE with early-return on cycle termination; non-smart rules zero cycle_start_equity and re-anchor the next non-skipped scan to a pinned wallet quantity; pin whether close_on_profit and a naturally-emptied cycle also terminate/re-anchor/re-arm.
- **REQ-PAR-004** _(must, P0)_ — EQUITY_DROP_PCT_SMART is one-shot per scan (smart_drawdown_fired), closes ONLY intrabar-losing positions (pin the valuation point), does a conditional re-anchor (pin the predicate/inequality), re-arms each non-skipped scan, and pin the cascade fall-through to survivors when a SMART fire does not terminate the cycle.
- **REQ-PAR-005** _(must, P0)_ — Equity-rule trigger basis pinned: EQUITY_DROP fires on min(close,intrabar adverse-extreme), EQUITY_RISE/close_on_profit on max(close,intrabar favorable-extreme); pin whether DROP measures drawdown from a fixed cycle_start_equity vs a ratcheted running peak.
- **REQ-PAR-006** _(must, cross)_ — Basket equity E that the equity cascade thresholds against = wallet + Σ open-uPnL WITHOUT subtracting Σ locked_margin (unlike sizing_capital, which does); E reads the POST-funding wallet on a boundary bar; pin the open-position uPnL entry-price basis (slipped fill vs un-slipped mark).
- **REQ-PAR-007** _(must, P0)_ — TRAILING_PROFIT state machine: clear peak when uPnL≤0; below activation preserve peak without triggering; peak from bar high(Buy)/low(Sell); trigger when per_unit<peak×0.5; the activation level derives from trailing_profit_pct on the same ROI-on-margin-at-leverage basis as TP/SL.
- **REQ-PAR-008** _(must, P0)_ — Time-rule order MR time_stop_minutes→MAX_DURATION→BREAKEVEN_TIMEOUT; pin each rule's age-clock reference point and boundary inclusivity; BREAKEVEN mutates ONLY TP (pin gross-entry vs net-of-fees), leaves SL and liq price unchanged, is skipped while trailing is active, and pin same-bar re-eligibility when trailing deactivates.
- **REQ-PAR-009** _(must, P0)_ — Wallet accounting: locked_margin is NEVER deducted at open (only entry_fee); a normal close adds wallet_delta; liquidation deducts locked_margin via a separate branch with NO exit fee and NO slippage.
- **REQ-PAR-010** _(must, P0)_ — Sizing qty=(sizing_capital·capital_pct/100·leverage)/entry_base_price floored to qty_step; reject qty<min_qty and required_margin>available but ADMIT at exact equality; pin the floored-to-exactly-0 disposition; flooring/rounding bit-identical to the Decimal oracle.
- **REQ-PAR-011** _(must, cross)_ — sizing_capital = running wallet + carried uPnL − Σ locked_margin, marked at last-close≤scan_time; immediate-mode depletes sequentially in abs(score) rank order (prior same-scan locked_margin + carried uPnL + entry_fee); batch-mode sizes every signal off the same scan-start capital with NO depletion; the carried-uPnL summation uses legacy insertion order.
- **REQ-PAR-012** _(must, P0)_ — skip_if_positions_open skips iff the open book is non-empty at scan START (latched at entry, so a mid-scan close does not un-skip it), still evaluates close rules, preserves the anchor, and does NOT re-arm smart_drawdown; an empty book, a zero-admitted (all-filtered) scan, and an empty (zero-signal) scan all take the normal admit + re-arm + re-anchor path.
- **REQ-PAR-013** _(must, P0)_ — Funding charged exactly once per 0/8/16h boundary via the (date,hour) dedupe regardless of candle granularity; longs pay/shorts receive (inverted for a negative funding_rate_fixed_pct); holding across N boundaries = N charges; zero when no boundary is crossed or funding_rate_model='none'; applied on open/close/liquidation/backtest_end boundary bars per legacy inclusivity.
- **REQ-PAR-014** _(must, P0)_ — Funding amount basis pinned bit-identically — notional price source (qty×entry vs qty×mark, slipped vs un-slipped) and settlement timing (settled to wallet at the boundary vs accrued in carried uPnL) — so a funding term is counted exactly once across both sizing_capital and basket equity and never double-counted in cross-scan sizing.
- **REQ-PAR-015** _(must, P0)_ — Price kernels pinned: TP/SL barriers from take_profit_pct/stop_loss_pct on ROI-on-margin-at-leverage with side sign; liq price from the fixed maintenance-margin assumption (never an instrument_info MM/risk-limit field); SL-clamp pulls a stop inside the liq band; slippage_bps applied adversely to entry/exit fill (sizing un-slipped; pin out-of-bar clamp); fee=notional×fee_rate_pct/100 both legs; pin tick-rounding order/direction, the sizing denominator representation, and the recorded liquidation exit_price.
- **REQ-PAR-016** _(must, P4)_ — A unit-level differential harness diffs the numba/SoA kernel reimplementations (sizing, TP/SL price, slippage, liq price, fee, trailing, breakeven, funding) against the pure-Python trading_rules helpers over a randomized input grid, and every raw config input (e.g. 0.1, 0.055) is coerced into the engine numeric domain identically across the Decimal and float64 paths.
- **REQ-PAR-017** _(must, P4)_ — Every Decimal quantization site (qty floor-truncate to qty_step, price/tick rounding, pnl/fee/funding rounding) is reproduced by the float64/numba path with the SAME rounding MODE and precision, enumerated site-by-site, and the legacy engine + oracle-regeneration Decimal context (prec + rounding) is explicitly set and hash-pinned.
- **REQ-PAR-018** _(must, cross)_ — Tolerance splits DISCRETE from CONTINUOUS: opened set, side, close_reason, exit-bar index, total_trades, and the selection set are BIT-IDENTICAL on every path; only money/price fields may differ within the <1% epsilon, with a documented per-field absolute floor for near-zero oracle values and signed-zero (−0.0==0.0) normalization in the comparator and persisted JSONB.
- **REQ-PAR-019** _(must, P4)_ — float64 must never flip a discrete decision near equality — barrier touch/no-touch, the “SL closer than liq” distance tie, and each filter threshold (score/age/drift) resolve identically to the Decimal oracle with the same `>=`/`>` boundary; NaN/None OHLC handled identically; cumulative accumulation drift stays below the smallest recorded-money quantum at the deepest (e.g. 200sym×365d) fixture.
- **REQ-PAR-020** _(must, P0)_ — Three-way reconciliation on EVERY fixture: trade.pnl=pnl−exit_fee−entry_fee−funding_paid AND Σtrade.pnl==metrics.net_profit==final_equity−starting_capital, accumulated in Decimal over recorded values (adding the per-trade sum `_assert_reconciles` lacks); holds with liquidation/negative-funding/force-close/basket-flatten terms combined, on DB read-back, and under bankruptcy (wallet<0); Σfees and Σfunding cross-check their metric keys; a zero-friction config yields exact gross PnL with clean zeros.
- **REQ-PAR-021** _(must, P0)_ — Structural count identity on every fixture: metrics.total_trades==len(trades)==wins+losses+breakevens, with every close_reason (take_profit, stop_loss, liquidation, equity_*, trailing_profit, breakeven_timeout, max_duration, mr_time_stop, backtest_end, pause_trading) present and none dropped/double-counted; total_trades is always present (incl. 0) so the UI never routes to the “no trades simulated” fallback; pin how a pnl==0 breakeven trade buckets into win_rate/profit_factor/streaks.
- **REQ-PAR-022** _(must, P0)_ — equity_curve pinned: seed a starting_capital anchor at the first scan's first signal_time (pin the fallback ts when the first scan is empty/all-filtered), append a point on every close, apply the stable None-first sort, append the terminal final_equity==wallet point only when >1e-9 from the last; force-close all positions at each symbol's last close in a deterministic, tie-broken order; verify via an ordered hash over the FULL stored JSONB (not the LTTB GET view); pin the per-point EquityPoint.drawdown_pct sequence and each point's ts source; freeze the zero-trade curve.
- **REQ-PAR-023** _(must, P0)_ — All ~45 summary metrics are frozen to the legacy formula (denominator, sign, breakeven inclusion) including degenerate sentinels (zero-trade, all-wins inf-guard, all-losses); MFE/MAE update every bar from that bar's high/low BEFORE exit checks, side-correct, from 5m extremes even when drilled, including same-bar and monotonic-trade edges; the in-memory trade-LIST order feeding streak metrics is pinned; the persisted BacktestTrade ordinal is contiguous/order-stable; any per-trade ROI field's denominator/sign is frozen.
- **REQ-PAR-024** _(must, cross)_ — Sharpe/Sortino reproduced under int64-epoch: the equity→returns derivation, inter-point time-delta periodization, a simulation_interval-aware annualization constant, and the risk-free/Sortino-MAR constants are pinned (hash-pinned constants); the SUMMARY max_drawdown_pct/_pct denominators (peak incl. the seed anchor vs starting_capital) are frozen; a full side×outcome sign matrix (4 quadrants + liquidation + funding-receive) freezes every money/excursion sign.
- **REQ-PAR-025** _(must, P0)_ — The 17-step filter chain order is preserved with a full per-signal disposition map (admitted-as-trade / filtered-at-step-S / signals_no_kline / qty-undersized / margin-rejected), pinning: min_score signed-vs-abs basis, confidence_filter ordinal ≥ inclusion, max_signal_age basis incl. ≤0 intra-scan ages, max_price_drift on the un-slipped entry_base_price, max_same_direction post-reverse count scope, equal-abs(score) intra-scan tie-break, same-symbol duplicate (differing + exact) handling, OPEN-book-scoped (not lifetime) existing-symbol dedup incl. cross-cohort, static whitelist∩blacklist precedence, per-scan vs lifetime counter increments, and fill_to_max_trades skip-rejected-and-continue vs stop-at-attempts.
- **REQ-PAR-026** _(must, P3)_ — The adaptive blacklist is a point-in-time, path-dependent set seeing exactly the trades closed before scan T (pin ≤T vs <T) within a 48h sliding window (enabled, min_trades=5, max_win_rate=30); the O(1) incremental win/total counter must equal the legacy full-history recompute, and which close_reasons (liquidation, backtest_end) feed win/total is pinned.
- **REQ-PAR-027** _(must, P0)_ — direction='reverse' flips the executed side via determine_side; signal_sides and the signed max_price_drift evaluate the RAW pre-reverse direction while max_same_direction counts the POST-reverse side; ALL side_sign accounting (funding direction, uPnL sign, MFE/MAE high-vs-low selection, TP/SL/liq barrier signs) is computed on the POST-reverse executed side, with a frozen reverse oracle and a pinned reverse∘MR composition order.
- **REQ-PAR-028** _(must, P0)_ — The mean_reversion route is reproduced bit-identically: mr_max_trades concurrent cap over ALL open positions (not generic max_trades), mr_capital_pct/mr_leverage/mr_tight_stop_pct sizing+stop overrides, mr_min_edge_pct/mr_target_capture_pct/mr_extreme_min_abs_score gates, the reference-mean formula (SMA/EMA, price field, mr_mean_period@mr_mean_interval, current-bar inclusivity), the target-capture exit and its pinned close_reason (added to the enum-coverage union), mr_short/mr_long_enabled side-gates, the mr_long_ack bypass, and mr_regime='ranging' gating; each trade records cohort-effective leverage/qty/strategy_kind; mixed-cohort caps and dedup compose per legacy.
- **REQ-PAR-029** _(must, P0)_ — Accepted/known divergences are preserved and asserted, never “corrected”: max_same_sector stays an engine no-op (service still emits max_same_sector_not_enforced), mr_long_ack is bypassed in backtest, strategy_cohort None defaults to trend, regime_staleness_minutes is live-only, and instrument_info / maintenance-margin / per-symbol max-leverage are shipped as the SAME snapshot the legacy engine used (no fresher snapshot, no leverage clamp).
- **REQ-PAR-030** _(must, P0)_ — Regime-ACTIVE behavior is frozen separately from the disabled no-op: the classifier label (regime_volatile_atr, regime_trend_ema_dist_pct over the per-scan ScanContext) and the gates that consume it (gate_session blocked/allowed hours, gate_btc_vol min/max threshold, routing to trend/none); toggling each feature OFF restores byte-identical 5m output.
- **REQ-PAR-031** _(must, P0)_ — target_goal lifetime early-stop halts at the exact legacy granularity for BOTH trade_count and profit_pct (pin the measured-equity basis, the evaluation timing, and mid-scan vs end-of-scan halt) opening neither one trade too many nor too few; the disposition of still-open positions at the halt (force-close at a pinned bar/price vs leave-open) and its terminal curve/reconciliation are pinned, as is the signals_no_kline effect on the per-scan/lifetime counters.
- **REQ-PAR-032** _(must, cross)_ — Engine + timed-pipeline determinism guards: ZERO wall-clock reads (only the cache completion frontier may call now()), RNG-free, no look-ahead (mutating any future-of-decision candle changes nothing beyond the entry-next-bar-open rule), no input mutation (two runs on the same objects identical and every input deep-equal to a pre-run copy), and byte-identical output under permuted PYTHONHASHSEED, permuted physical DB row order, permuted config symbol-list order, a constant time-translation of all epochs, and a non-UTC/DST host timezone.
- **REQ-PAR-033** _(must, P3)_ — The SoA global-timeline merge-walk + searchsorted reproduce the legacy per-symbol linear scan byte-for-byte: scans iterate in STABLE first-signal_time order with the legacy tie-break; per-symbol pointers never reset; “last close≤T” uses side='right'−1 (last duplicate epoch) and entry-fill uses side='left' (first duplicate epoch); misaligned/gapped/staggered-listing timelines and a pinned no-prior-mark fallback for open positions are handled; repeated equal-(symbol,time) queries are idempotent; signal_time is normalized to Bybit-ms/UTC matching kline open_time before lookup; the open-book Σ-uPnL is summed left-to-right sequentially (NOT numpy pairwise), and prange parallelizes ONLY the outer sweep.
- **REQ-PAR-034** _(must, P0)_ — Same-bar eligibility is pinned: a position may exit in its OWN entry-fill bar exactly as legacy (entry=next-bar-open; [low,high] spanning a barrier, a gap-open beyond the barrier, and immediate same-bar liquidation), and whether a freshly-opened position joins that bar's equity cascade / SMART selective-close set / trailing-peak seed; a position opened on the terminal bar and same-bar backtest_end force-closed is reproduced bit-identically.
- **REQ-PAR-035** _(must, cross)_ — Golden NO-OP guarantee: with empty instrument_info, empty scan_contexts, empty fine_klines and regime disabled, engine output is byte-identical to the pure 5m no-drill path — including the instrument_info-absent qty_step/min_qty default and the present-but-zero qty_step/tick_size guard — and is re-run after EVERY phase.
- **REQ-PAR-036** _(must, P1)_ — Cache coverage parity: candle_count is a FACT column only and never appears in a fetch-decision predicate (a sealed short day, e.g. 144/288, triggers zero refetch); closed candles are fetch-once-ever with listing/current-day partial-day clipping preserved; a content_sha256 per sealed day permits refetch ONLY on an explicit hash-mismatch escape hatch; store_klines stays ON CONFLICT DO NOTHING and _update_coverage stays GREATEST-monotonic (manifest added as migration v58 ADD COLUMN); the refreshable (always-fresh) set is exactly the forming day at open_time≥frontier.
- **REQ-PAR-037** _(should, P5)_ — Columnar/memo read identity: any LRU/memo/Parquet/DuckDB/Arrow cache hit returns klines byte-identical (same epoch ints, same OHLCV float bytes, same ordering) to a fresh Postgres row read, gated by a dedicated test before that path is enabled.
- **REQ-PAR-038** _(must, P5)_ — Coarse-interval data-source determinism: when both exchange-native coarse rows and derive-from-fine are available, the read path picks ONE documented authoritative source deterministically (a 15m/1h/4h backtest returns identical klines/trades run-to-run); fine→coarse derivation is bit-identical across DuckDB/Polars/pandas backends (left-closed, UTC-aligned, identical OHLCV aggregation) including a documented fine-gap rule; full-engine golden fixtures are frozen for 15m/1h/4h.
- **REQ-PAR-039** _(must, P2)_ — Batched loaders are decision-identical: the kline ANY($1) load buckets into strictly-ascending per-symbol arrays byte-identical to a per-symbol ORDER BY read, and the batched scan_source signal + ScanContext load is byte-identical (every field, scan grouping, stable first-signal-time order) to the legacy per-scan load; duplicate/overlapping scan rows are handled per legacy (no double-anchor/re-arm).
- **REQ-PAR-040** _(should, P6)_ — Sentinel “disable” config values are truly inert in the SEQUENTIAL engine, not merely a fast-path eligibility gate: max_drawdown_pct==100 never fires EQUITY_DROP, off-sentinel stop_loss/take_profit/trailing never fire — byte-identical to a config omitting them — and any vectorized barrier fast-path is provably gated to configs where it equals the sequential engine.
- **REQ-PAR-041** _(should, cross)_ — A mid-run failure of an accelerated path (numba/DuckDB/Parquet) restarts the WHOLE run clean on the pure-Python/Postgres path (never continue-from-failure splicing two engines' partial latch/equity state), and the recovered result is byte-identical to the oracle.
- **REQ-PAR-042** _(must, cross)_ — The full-scale 90d×50sym×5m drilldown-OFF benchmark carries its own frozen result fingerprint (content hash over ordered trades + ordered equity_curve + all ~45 metrics) asserted byte-identical across Phases 0–6, so no phase buys a speedup by changing results at scale; the optimized run must finish well under the 120s hard wall-clock cap.
- **REQ-PAR-043** _(must, cross)_ — Wrapper parity: the POST /backtest service pipeline (scan_source load + filter chain + persist) equals a direct in-process engine call (discrete-identical / continuous within epsilon); a single sweep combo's persisted result equals a standalone backtest_run of that exact config; optimize_config's PROPOSED config reproduces the per-combo metrics it was ranked on; and re-running a config across the dark→optimized flag flip stays within tolerance with identical discrete outcomes.
- **REQ-PAR-044** _(must, P5)_ — Fingerprint & persistence stability: content_sha256 is a pure function of the canonical candle bytes, independent of engine git SHA or optimization-flag generation (concurrent rolling-deploy versions converge to one hash, no spurious refetch); the equity_curve manifest hash and sweep config_hash are invariant to net-new sibling result columns; every recorded numeric value survives the float→NUMERIC/JSONB round-trip losslessly with the three-way reconciliation holding on reloaded values.
- **REQ-PAR-045** _(must, cross)_ — The Buy&Hold BTC baseline is reproduced on a fully isolated path (legacy first-vs-last-close anchors) and EXCLUDED from the Σtrade.pnl reconciliation even when BTC/USDT is itself traded; auxiliary series (B&H, btc_vol, MR-mean) are valued off the SAME forming-day snapshot the engine consumed; degenerate/malformed scan_source fields (≤0/NaN/NULL price, missing side, out-of-enum confidence) and contradictory barrier configs are dispositioned byte-identically without raising; reproduce/retry detects pruned scan_source signals and reports ‘unavailable’ rather than silently diverging; and an open position on a symbol whose klines end mid-window is force-closed at THAT symbol's last candle, never the global window end.

## Cache [CACHE] — 50 items

- **REQ-CACHE-001** _(must, P1)_ — A warm identical rerun (same config, same session) completes <2s with 0 Bybit calls, 0 kline SELECTs, and 0 Postgres writes (no store_klines, no seal upserts, no GREATEST coverage upserts), asserted by separate read/write counters.
- **REQ-CACHE-002** _(must, P1)_ — A cold run bounds Bybit kline requests to <=ceil(true_missing_candles/1000), never refetches a sealed day; cold wall-time scales with missing candles, not total history (fixes RC-4 span-collapse + RC-3).
- **REQ-CACHE-003** _(must, cross)_ — A cold run and a subsequent warm run of the same config produce byte-identical trades, ordered equity_curve, and ALL metrics (incl. total_trades) across LRU- and columnar-served reads; cache changes WHERE not WHAT.
- **REQ-CACHE-004** _(must, P1)_ — Because the data cache key excludes strategy params, a param-tweak rerun over the same (symbols,interval,window) issues 0 Bybit + 0 kline SELECTs; a symbol add/remove rerun loads only the delta symbols (0 data I/O for unchanged).
- **REQ-CACHE-005** _(must, P1)_ — Completion frontier = floor(now_utc/T)*T computed per (symbol,interval), recomputed at read time each run (auto-seals later) yet held stable within a single run; boundary exact: open_time==frontier-T sealed, ==frontier forming.
- **REQ-CACHE-006** _(should, P1)_ — Frontier applies an explicit skew-margin safety lag, is a monotonic ratchet (a backward clock step never un-seals), and derives now_utc from one coordinated authoritative source so instances agree on the boundary day's class.
- **REQ-CACHE-007** _(must, P1)_ — A day seals iff day.end<=frontier AND a post-frontier HTTP-200 success confirms its terminal candle; pre-frontier or any failed/partial/timeout/429 response never seals (stays unsealed for retry, partial rows stored idempotently, bounded backoff, actionable error).
- **REQ-CACHE-008** _(must, P1)_ — Refetch predicate is exactly need_fetch=(not sealed) AND day_overlaps([listing,delist]) AND day.end>last_covered with NO candle-count term; a 144/288 short sealed day is never refetched while the forming day (date>=today-1) still refreshes.
- **REQ-CACHE-009** _(must, P1)_ — Per-day manifest row keyed (symbol,interval,day_utc) persists sealed, first_open_ts, last_open_ts, candle_count, gap_count, listing_snapped, delisted, content_sha256, fetched_at; once sealed never re-evaluated/refetched regardless of count.
- **REQ-CACHE-010** _(must, P1)_ — ensure_coverage is gated by a manifest 'range complete' check (SSOT, never a live COUNT(*)); seal-state for all symbols resolves in 1 batched ANY($1)...day_utc BETWEEN query, index-backed so latency is O(range days), independent of manifest size.
- **REQ-CACHE-011** _(must, P1)_ — _load_klines issues exactly 1 Postgres query per interval for all symbols via WHERE symbol=ANY($1) (bucketed in Python), independent of symbol count (assert query_count==1) — fixes RC-6 N+1.
- **REQ-CACHE-012** _(must, cross)_ — Per-day manifest stays consistent with the legacy GREATEST coverage HWM (never sealed=true while HWM reports uncovered; both classify ranges identically); a v58 sealed-column NULL row is treated as UNSEALED/refetchable, never sealed=true, never an error.
- **REQ-CACHE-013** _(must, P1)_ — A closed day with existing kline_cache rows but no manifest seal (sealed IS NULL) is lazily sealed from the Postgres rows with 0 Bybit refetch (runtime equivalent of the v58 backfill).
- **REQ-CACHE-014** _(should, P1)_ — Runtime seal writes batch into O(1)/O(symbols) round-trips (executemany/INSERT...ON CONFLICT), a day entering the batch only after rows are durable in all tiers; seal-row and gap-range writes are race-safe idempotent upserts converging under >=3 concurrent writers.
- **REQ-CACHE-015** _(must, P1)_ — REST page size raised 200->1000 (Bybit cap) so gap-fill request count for a fixed span is ~5x lower (assert ratio ~5x) and spans >3.47 days stop truncating to the most-recent 1000.
- **REQ-CACHE-016** _(must, P1)_ — Fetcher issues one span per contiguous gap-run (never a single min..max span) and paginates each run to FULL coverage in one ensure_coverage call (raise/remove _MAX_PAGES=5 or loop until last_open>=gap_run_end) so interior candles are never left perpetually missing.
- **REQ-CACHE-017** _(must, P1)_ — Ingest normalizes Bybit descending pages to ascending before store/sha, stitches seams so each grid slot is fetched exactly once (no drop/dup, correct endpoint semantics); content_sha256 is seam-invariant and two independent cold runs seal an identical (sym,interval,day) set/class/sha.
- **REQ-CACHE-018** _(must, P1)_ — Returned candles are clipped to the in-life window before store/sha (drop pre-start, open_time>=frontier forming, or outside [listing,delist]); before sealing, any pre-existing stale forming row (DO-NOTHING) is overwritten so staleness is never frozen into a sealed day.
- **REQ-CACHE-019** _(must, cross)_ — A multi-day gap-run failing partway seals every fully-covered closed day already stored, leaving only the remainder unsealed; the pagination loop checks the cooperative cancel and 120s deadline between pages, aborting cleanly with partial progress persisted and a retryable state.
- **REQ-CACHE-020** _(must, P1)_ — Per-symbol load bounded to [min(signal)-lookback, max(signal)+max_hold], lookback covering every indicator window (adaptive_blacklist 48h, mr_mean_period*interval, btc_vol, regime EMA), snapped outward to whole UTC days for seal lookups; results byte-identical to a full-range oracle (same signals_no_kline + opened-trade set).
- **REQ-CACHE-021** _(must, P2)_ — Loaded symbol set equals distinct(scan_source signal symbols) UNION required aux symbols (BTC for B&H/btc_vol, MR-mean ref) exactly — no signaled symbol omitted (silent signals_no_kline), no extraneous symbol loaded.
- **REQ-CACHE-022** _(must, P1)_ — All auxiliary kline paths (Buy&Hold BTC separate loader, btc_vol BTC, MR-mean) route through the sealed-manifest/LRU cache so a warm rerun issues 0 Bybit calls across ALL of them; B&H keeps return/excess within <1% tolerance and nulls its keys (never aborts _persist_results) on fetch failure.
- **REQ-CACHE-023** _(should, P2)_ — Coarse-interval and auxiliary series are derived from the fine 5m sealed base via resample, not a separate coarse fetch: warm 5m->1h issues 0 Bybit + 0 kline SELECTs; even cold, a coarse run fetches only the fine base once.
- **REQ-CACHE-024** _(should, P2)_ — Within a session/sweep: fine->coarse resample, indicator/feature series (EMA/ATR/btc_vol/MR-mean), and scan_source signals+ScanContexts are each memoized once per (symbol,interval,range) excluding strategy params, so threshold/param-varying combos reuse them (0 re-derivation, 0 signal/context queries on rerun); memos bounded and byte-identical.
- **REQ-CACHE-025** _(should, P1)_ — instrument_info (qty_step,min_qty,tick_size) and symbol_lifecycle exchange lookups are cached (TTL/sealed snapshot) so a warm rerun's total Bybit call_count==0 across kline+instrument_info+lifecycle paths, not just the kline loaders.
