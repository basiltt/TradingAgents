# TradingAgents Data Source Tools & Architecture Analysis

## QUICK SUMMARY

1. DATA SOURCES AVAILABLE:
   - CRYPTO: Only Bybit (perpetual futures: klines, indicators, funding rates, OI, ticker)
   - STOCK: yfinance + Alpha Vantage (dual vendor with fallback)
   - FUNDAMENTALS: Balance sheets, cash flow, income statements
   - NEWS: Stock (yfinance + AV) + Crypto (via coin names)
   - INSIDER: Alpha Vantage only

2. ANALYST TYPES:
   - STOCK: 4 analysts (market, social, news, fundamentals)
   - CRYPTO: 3 analysts (crypto_technical, crypto_derivatives, crypto_news)
   - Tool pattern: LangChain @tool wrappers for stock
   - Factory pattern: make_crypto_tools() for crypto (shared cache/limiter)

3. FRONTEND FLOW (ConfigForm.tsx):
   - Line 22-23: Define STOCK_ANALYSTS, CRYPTO_ANALYSTS lists
   - Line 215-216: Dynamic selection based on asset type toggle
   - Line 333-336: Reset analysts when toggling stock/crypto
   - Submit → backend with selected analysts + asset_type

4. REUSABILITY FOR CRYPTO FUNDAMENTALS:
   - Can reuse: News tools (Alpha Vantage NEWS_SENTIMENT covers crypto), make_crypto_tools() factory
   - Cannot reuse: Stock-specific fundamental APIs (no crypto alternatives)
   - Need to build: On-chain data (Glassnode), social sentiment (Reddit), project metrics (Messari)

5. GAPS:
   - No on-chain data sources (whale wallets, holder distribution)
   - No crypto-native social sentiment (Reddit, Twitter, Discord)
   - Limited crypto news (only via Alpha Vantage topics, not CryptoPanic)
   - No fallback for Bybit (single source for crypto)

## DETAILED FINDINGS

### BYBIT CRYPTO TOOLS
- Klines: 15min, 60min, 240min, Daily
- Indicators: RSI, MACD, Bollinger Bands, EMA
- Funding rates: Historical rates
- Open interest: Multiple timeframes (5min, 15min, 30min, 1h, 4h, 1d)
- Ticker: Current price snapshot
- Features: Rate limiter (80 capacity, 16/sec refill), circuit breaker, output truncation (50KB)
- Non-critical tools: Funding rates, OI, ticker return "unavailable" gracefully on failure

### STOCK TOOLS PATTERN
@tool decorator → route_to_vendor() → vendor implementation
- get_stock_data(ticker, start, end)
- get_indicators(ticker, interval)
- get_fundamentals(ticker, curr_date)
- get_balance_sheet(ticker, freq, curr_date)
- get_cashflow(ticker, freq, curr_date)
- get_income_statement(ticker, freq, curr_date)
- get_news(ticker, start_date, end_date)
- get_global_news(curr_date, look_back_days)
- get_insider_transactions(ticker)

All return HTML-escaped, truncated strings (max 50KB)

### CRYPTO TOOLS FACTORY
make_crypto_tools(cache, limiter, circuit_breaker, api_key, api_secret) → list of @tool
- Shared cache/limiter/circuit_breaker across all tools
- Tool filtering by name in analyst creation
- Non-critical tools fail gracefully
- All tools take datetime strings (converted to milliseconds internally)

### ANALYST SELECTION FLOW
1. Frontend ConfigForm.tsx detects asset_type toggle (stock/crypto)
2. Sets activeAnalysts = isCrypto ? CRYPTO_ANALYSTS : STOCK_ANALYSTS
3. Renders checkboxes for available analysts
4. Form submission passes analysts array to backend
5. Backend graph setup: setup_graph(selected_analysts) or setup_crypto_graph(selected_analysts)
6. Graph dynamically creates nodes for selected analysts only
7. Analysts execute sequentially, passing reports to next analyst

### VENDOR CONFIGURATION
- interface.py defines VENDOR_METHODS mapping
- Config file sets data_vendors per category
- Routing: check tool-level override → category-level → vendor list order
- Fallback: Only on AlphaVantageRateLimitError (yfinance → alpha_vantage)
- Frontend: localStorage persists vendor selection across sessions

### CRYPTO ANALYST DIFFERENCES
- Tools passed as parameter (not global)
- Tool filtering by name (selective use, not all available)
- News analyst reuses stock get_news() with coin name as ticker
- Derivatives analyst uses funding rates/OI (crypto-specific)
- No multi-analyst debate after analysts (goes straight to Trader)
- Only 2-way risk debate (Bull ⟷ Bear) vs. 3-way for stocks
