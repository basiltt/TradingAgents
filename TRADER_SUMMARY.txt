COMPREHENSIVE TRADER AGENT ANALYSIS
=====================================

LOCATION: tradingagents/agents/trader/trader.py

WHAT IT DOES:
- Converts Research Manager's strategic investment plan into concrete transaction proposal
- Provides specific entry/exit prices, position sizing, risk parameters
- Acts as tactical translation between strategy and execution

INPUTS RECEIVED:
1. company_of_interest: Ticker (AAPL, BTC.SPOT)
2. investment_plan: Research Manager output (Buy/Hold/Sell + rationale)
3. market_report: Technical analysis (SMAs, MACD, RSI, Bollinger Bands, ATR)
4. sentiment_report: Social media signals (bullish %, retail vs institutional)
5. news_report: Catalysts, insider transactions
6. fundamentals_report: P/E, debt, cash flow, earnings, growth
7. messages: LangGraph message history

ANALYSIS PERFORMED:
1. Instrument context building (preserve ticker with exchange suffix)
2. Investment plan parsing (extract recommendation, rationale)
3. Multi-source synthesis (align signals from 4 analysts)
4. Direction determination (Buy/Hold/Sell)
5. Confidence scoring (1-10 based on signal alignment)
6. Entry/exit level calculation (entry, stops, take-profits)
7. Risk/reward ratio computation (typically 1.5:1 to 3:1)
8. Position sizing (1-5% portfolio, volatility-adjusted)
9. Time horizon estimation (days to 12 months)

SYSTEM PROMPT:
"You are a trading agent. Anchor reasoning in analysts' reports and research plan.
Provide: confidence score (1-10), entry price, take-profits (1-3 levels), stop-loss,
risk/reward ratio, position sizing, time horizon."

OUTPUTS PRODUCED:
1. messages: LangGraph message container
2. trader_investment_plan: Rendered markdown proposal
3. _trader_signal_data: Typed TraderProposal object (internal-only)
4. sender: "Trader" for routing

OUTPUT SCHEMA (TraderProposal):
- action: Buy|Hold|Sell (required)
- confidence: 1-10 (optional)
- reasoning: 2-4 sentences (required)
- entry_price: float (optional)
- stop_loss: float (optional)
- stop_loss_2: secondary stop (optional)
- take_profit_1-3: exit targets (optional)
- risk_reward_ratio: float (optional)
- position_sizing: string (optional)
- time_horizon: string (optional)

GRAPH POSITIONING:
Analysts (parallel) -> Bull/Bear Debate -> Research Manager
  -> TRADER <- consumes investment_plan
  -> Risk Analysts (debate risk/reward)
  -> Portfolio Manager (final decision)

EDGES:
Research Manager -> Trader
Trader -> Aggressive Analyst

DOWNSTREAM CONSUMPTION:
- Risk Analysts: Read trader_investment_plan for debate
- Portfolio Manager: Synthesizes both plans into final decision

STRUCTURED OUTPUT:
Provider-specific modes:
- OpenAI/xAI: json_schema
- Gemini: response_schema
- Anthropic: tool-use
- Ollama: Free-text fallback

Graceful fallback on failure:
structured_call fails -> retry free-text -> return (plain_text, None)

KEY DIFFERENCES:
Research Manager: Strategic (buy/hold/sell conviction)
Trader: Tactical (specific entry $155.30, stop $152.50, TP $158.50)
Portfolio Manager: Execution (final 5-tier rating + risk adjustment)

DESIGN PATTERNS:
1. Instrument context preservation (preserve tickers)
2. Rendering for downstream (markdown + typed objects)
3. Optional fields (support various scenarios)
4. Internal signal carriers (_trader_signal_data)
5. Factory pattern with partial application

SUMMARY:
Trader is the tactical translation layer between strategy and execution.
Consumes: Investment plan + 4 analyst reports
Performs: Signal synthesis + confidence scoring + level calculation
Produces: Transaction proposal with specific prices/sizing
Integrates: LangGraph with structured + fallback
Feeds: Risk analysts + Portfolio Manager
